NPE
- java 용어 이며, 프로그램이 실행되는 도중에 값이 null인 상태에서 객체에 접근하려고 할 때 발생하는 오류이다.
- 옵젝씨에서는 변수나 프로퍼티에 `nil` 값을 할당 할 수 있었고, 이  nil 값이 객체에 대한 참조로 사용 되면 런타임 중에 애플리케이션 크래시가 발생할 수 있었습니다. 이와 같은 상황은 NPE와 비슷한 상황이며 값이 없는 상태에서 객체에 접근하려고 할 때 발생 하는 문제 이다.
- NPE는 프로그램에서 null값으로 인해 객체에 접근할 수 없는 상황을 이다. Objective-C에서도 nill값을 가진 객체에 접근하면 유사한 런타임 오류가 발생 할 수 있다. Swift에서는 옵셔널 개념을 도입하여 문제를 사전에 방지하고 코드 안정성을 강화했다.
 
Swift에서 옵서녈 개념이 도입된 이유 중 하나는 이런 NPE 문제를 방지하고 코드 안전성을 향상 시키기 위해서 이다.
Swift에서 변수나 프로퍼티 값이 nil 일 수 있는 경우 해당 값을 옵셔널이며 컴파일러가 값이 있는지 없는지 체크하도록하며, 값이 없을 경우에 대비한 안전한 코드 작성을 강제하여 런타임 시에 NPE와 유사한 문제를 사전에 방지 할 수 있게 된다.

추가적?
<br>
Objective-C와 Swift는 둘 다 Apple이 개발한 프로그래밍 언어로, iOS 및 macOS 애플리케이션 개발에 사용됩니다. 그러나 두 언어는 언어적 특성과 기능 면에서 상당한 차이가 있습니다. 옵셔널(Optionals)이라는 개념도 두 언어에서 다르게 다루어집니다.
 		Objective-C:
 		Objective-C에서는 변수나 프로퍼티에 nil 값을 할당할 수 있습니다. 이는 해당 변수가 값이 없음을 나타냅니다. 그러나 이것은 컴파일러나 런타임에서 강제적으로 체크되지 않습니다. 따라서 nil 값이 할당된 변수에 접근하면 애플리케이션은 크래시될 수 있습니다. 이는 Objective-C 코드가 런타임 중에 안정성 문제를 일으킬 수 있는 가능성을 내포하고 있다는 의미입니다.
 		Swift:
 		Swift는 타입 안정성과 코드 안정성을 강조하는 언어로, 옵셔널이라는 기능을 통해 값이 없을 수 있는 변수를 더 안전하게 다룰 수 있도록 지원합니다. Swift에서는 변수나 프로퍼티의 값이 nil일 수 있는 경우, 해당 변수의 타입에 물음표(?)를 붙여 옵셔널로 선언합니다. 이를 통해 컴파일러가 컴파일 시점에서 옵셔널 타입의 변수 사용을 체크하고, nil일 경우에 대비한 안전한 코드 작성을 요구합니다.

```swift
 var optionalString: String? = "Hello, Swift"
  
 // 옵셔널 체이닝을 통한 안전한 접근
 if let unwrappedString = optionalString {
     print(unwrappedString)
 } else {
     print("String is nil")
 }
```

* 위 코드에서 optionalString 변수는 옵셔널로 선언되었습니다. 만약 이 변수가 nil이 아니라면, 옵셔널 바인딩을 통해 안전하게 값을 언래핑하고 사용할 수 있습니다.
* 요약하자면, Swift는 타입 안정성과 코드 안정성을 강조하는 언어로서, 옵셔널을 통해 값이 없을 수 있는 상황을 더욱 안전하게 처리할 수 있도록 지원합니다. Objective-C는 이에 비해 옵셔널을 제공하지 않거나 덜 강제적으로 다루기 때문에, Swift에서는 옵셔널을 통해 보다 예측 가능하고 안전한 코드를 작성할 수 있습니다.
